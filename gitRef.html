Git Tutorial

In order to use git, you need to use command prompt or powershell. I will be using command prompt.

On the image, you can see I used c: to change to the c drive, and then cd (change drive) \Program Files (x86)\Ampps\www\CSC245, then cd\ Week2Assignment to go to folder I wanted.

I then used git Clone https://github.com/JLessar/Week2Assignment.git to copy the repository I previously created at GitHub. I was able to get the proper address by copying the yellow highlighted text.

Next I will pull the code in, to make sure I have the latest version, using git pull https://github.com/JLessar/Week2Assignment.git

I will also use git status to see the changes I made. Unsaved changes appear in red.

These changes need to be saved using git add .. Using git status again, we can see the red has changed to green, meaning we are ready to upload them to the repository.

Using git commit -m" " I commit to the changes and can add commentting between the quotations. This is key, because it will show up in your repository for all to see.

Lastly, I will push the freshly updated files into the GitHub repository. This is done by using the git push command.

You can now see the updated, with comments, GitHub repository.

With the comments we added using commit -m" " visible on all the files modified.

You can view all the commits by clicking the commits, which will display the list of what and when was committed.

You can even create different branches from the master, so you can keep different revisons. I created one called tempBranch.

Anything you do can be compared to the Original Master branch. You can even merge them.



Git Cheat Sheet

git Clone (website/file address) This command is needed only once to download the file which you will be working on. It will download and set up the files, based on the folder structure in gitHub.

dir This command shows the directory/files in the folder.

dir /ah This command will show ALL the files, even the hidden ones.

git Pull (website/file address) This command pulls down the current files from gitHub.

git Status This command shows the changes you made on your localhost.

git add . This command updates and saves the changes made on your localhost.

git Commit -m"This is the comment section" This command is used to comment the changes and what was updated, it is key to enter in a detailed account of your revisions. This will help you and your team track what was added and/or modified.

git Push The command pushes or adds your changes to the gitHub file, creating a new updated file. This file is also carries your comments on the additions you made.
